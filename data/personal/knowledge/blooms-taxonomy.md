# How I use Bloom's Taxonomy

This file answers questions about Bloom's Taxonomy, Taksonomi Bloom, cognitive levels, learning objective verbs, assessment alignment.

## Quick answer

I use the **revised Bloom's Taxonomy** (Anderson & Krathwohl, 2001) to write learning objectives at the right cognitive level, then design assessments that match. Six levels — Remember, Understand, Apply, Analyze, Evaluate, Create — give me a shared vocabulary with SMEs and a sanity check on whether a module is teaching at the right depth.

## How I apply it

* **Match objectives to job-level operation, not what's easy to teach.** Modal Cycle Zero needed Field Officers to *Apply* in client conversations and *Analyze* objections in real time — so objectives target Apply/Analyze, not Remember.
* **Align assessment to the objective's level.** Apply-level objective → scenario simulation, not a definition multiple-choice. Anti Harassment quizzes ask learners to *classify* situations, not just define harassment.
* **Use it to validate scope.** When SMEs over-stuff a module I push back: *"Everything here is at Remember level — should this audience be operating at Apply by now?"* Reanchors the conversation on outcomes, not content volume.
* **Cognitive × Knowledge matrix when scope is unclear.** Cross the 6 cognitive levels with 4 knowledge types (Factual, Conceptual, Procedural, Metacognitive). For complex curricula like Amartha System Architecture, I map each topic to a cell — catches gaps where a module is heavy on Factual but light on the Procedural skill the learner needs.

## What I avoid

* Verb-list cargo-culting — "Understand X" is fine if the assessment really tests understanding.
* Climbing the pyramid for its own sake — not every module needs Create level.
* For soft-skill training (Modal Cycle Zero communication) I use Krathwohl's **affective domain** or Dreyfus's **skill-acquisition stages** instead.
