# How I apply ADDIE

This file answers questions about ADDIE, instructional design process, training development phases, evaluation frameworks.

## Quick answer

I use **ADDIE** (Analyze → Design → Develop → Implement → Evaluate) as a backbone for any non-trivial training project, but I run it iteratively, not waterfall-style. Each phase produces a concrete artifact the next phase consumes — so SMEs and stakeholders sign off step-by-step instead of waiting until launch.

## How each phase looks

* **A — Analyze.** Map audience, performance gap, and constraints before opening Storyline. I push back on training-as-default — sometimes the answer is a job aid, not a course. For **Modal Cycle Zero**, Analyze surfaced that FOs needed objection-handling under pressure, which reshaped the curriculum away from feature-list toward simulation-heavy practice.
* **D — Design.** Curriculum map (competencies → objectives → topics → assessment items, all aligned), storyboards, and a pre/post + N-Gain assessment blueprint. Bloom's Taxonomy lives here. Lecture scripts are split from simulation scripts so each segment hits its intended cognitive level.
* **D — Develop.** Articulate Storyline / Rise, motion graphics, Figma assets. I prototype early so SMEs sanity-check storyboards before full production burn.
* **I — Implement.** Deploy as SCORM to Moodle LMS, coordinate rollout across regions, ensure devices are tested.
* **E — Evaluate.** Where most "ADDIE in name only" projects fail. I bake evaluation into Design from day one: **Reaction (Kirkpatrick L1)** post-survey, **Learning (L2)** pre/post + N-Gain, **Behavior (L3)** follow-up where possible.

## Concrete outcomes (validated via paired pre/post)

* **Modal Cycle Zero** — N-Gain **44.63%** (Medium Gain) across the field team.
* **Training Client Protection** — scores **54.04 → 79.12**, N-Gain **44.64%** (Medium Gain).
* **Dunia Geometri** — math achievement **46 → 76** (+30 points), **84.6%** measured learning interest.

## How I run it in practice — not waterfall

* Analyze + Design overlap (prototype objectives + storyboards in parallel).
* Develop runs in **vertical slices**: one full module end-to-end before scaling, so quality issues surface fast.
* Evaluate happens **mid-project** too — pilot one module to a sample group before producing the rest.
* For tight-timeline AI/internal work I lean toward **SAM (Successive Approximation Model)** and use ADDIE artifacts as deliverables inside SAM cycles instead of as gates.
